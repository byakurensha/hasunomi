HASUNOMI 1.4.0

- Added a simple Japanese six-up receipt template as editable ODG.
- One A4 landscape sheet is divided exactly into six 148.5 x 70 mm receipts.
- Fixed cut lines use x=148.5 mm, y=70 mm, and y=140 mm; content stays at
  least 5 mm inside each edge.
- The template contains only issue date, recipient, amount, description,
  receipt statement, issuer name, and optional issuer address/contact.
- Receipt values can be entered in JSON or left blank for handwriting.
- Printing requires 100 percent scale without fit-to-page. PDF remains
  optional and is created only when explicitly requested.

HASUNOMI 1.3.0

- Added a separate seven-stage Repair Process for structural consolidation
  after Improvement finds duplication, contradiction, retired routes, or
  growth in always-loaded guidance.
- Added `repair-check` and `repair-done`. The check reuses doctor, verifies
  required Skills and unique Skill identities, rejects duplicated substantial
  always-loaded guidance and retired routes, and reports review-only size
  metrics without authorizing automatic deletion.
- Improvement now ends with a lightweight Repair gate instead of expanding
  role or rule files by default.
- Re-audited the complete public and private General surfaces. Corrected the
  last A4 reference to a retired General-root OUTBOX, removed duplicate private
  OCR verification, and stopped Lifesaver from creating an OUTBOX report for
  every diagnosis.
- Repair remains on demand and adds no daemon, scheduler, startup scan, second
  memory, or fixed recurring task.

- Major Markdown checkpoints are now fully automatic. Project completion
  commits the active project immediately, and the next Claude Code session
  start quietly records any remaining allowed Markdown edits, renames,
  deletions, and new project MASTER files.
- Automatic checkpoints still obey the Markdown allowlist. Materials, Output,
  Archives, binary files, and general text files remain backup-owned.
- Staged renames are recorded as complete old-path deletion and new-path
  addition pairs, including changes already prepared by VS Code Source Control.
- VS Code Source Control no longer requires routine user operation.

- Limited workspace Git history to the main Markdown operating files,
  reusable Memory, and each project MASTER.md.
- Materials, Output, Archives, general text files, and binary files now remain
  backup-owned and cannot enter routine project checkpoints.
- Setup removes previously tracked excluded files from the Git index without
  deleting the working files.

- Reduced routine management overhead without removing project, Memory,
  Improvement, Git, document, OCR, delivery, or recovery capabilities.
- Formal Improvement now runs by explicit request or evidence-backed value
  judgment instead of a fixed completed-work count.
- Project checkpoints commit only the active project and review evidence, so
  unrelated projects and user edits are not swept into the same Git commit.
- Clarified that cloud delivery is optional and environment-owned; the public
  pack contains no bundled cloud transport or connector instruction.
- Setup now names exact validation commands so Claude does not inspect bundled
  tool source merely to discover routine command syntax.
- SHAREBOX is now one flat verified queue for cross-environment and
  cross-session handoff. It supports explicit send and receive, consumes a
  source only after verified intake, and uses handoff Markdown instead of
  direction folders when routing context is needed.

- Project folder names now preserve the user's active language instead of
  requiring lowercase ASCII English slugs.
- Japanese project names remain Japanese and are normalized only for unsafe
  separators; they are not translated for filesystem convenience.

- Fixed the supplied A4 ODT renderer for Japanese documents.
- Japanese title, heading, and table-header text now carries explicit Asian
  font family, size, and weight attributes.
- Added three heading levels, compact table paragraph styles, complete header
  borders and padding, and content-aware table column widths.
- Added a recipient-style ODT structure test for these guarantees.

- First stable public release.
- Uses the official Claude Code VS Code extension and session history as the
  user-facing workspace.
- Includes durable project records, Desktop OUTBOX and verified SHAREBOX,
  Memory, formal Improvement, Git text checkpoints, recovery, updates, OCR,
  editable A4 ODT, and editable Photo Slide ODP workflows.
- Keeps cloud delivery, private runtimes, provider credentials, and
  organization-specific integrations outside the public core.

- Removed the bundled cloud-delivery integration from the public pack. Cloud
  delivery now belongs to the user's provider or existing environment;
  Desktop OUTBOX and verified SHAREBOX remain available.

- Added the supplied Photo Slide template: editable 16:9 ODP, title, rule,
  page number, uncropped phone photos, and six one- or two-photo layouts.
- Added LibreOffice render verification for actual Noto Sans CJK JP Regular
  and Bold embedding; OS font registration alone no longer passes setup.

- Rebuilt around the official Claude Code VS Code session history.
- Users stay in the General workspace; internal projects are created quietly.
- Removed COMMON, manual project switching, duplicate skills, and hidden
  HASUNOMI setup state.
- Projects now contain only MASTER.md, Materials, Output, and Archives.
- Human delivery now uses Desktop OUTBOX. General-local Inbox and Outbox are
  retired.
- Added a generic Desktop SHAREBOX handoff, later expanded to verified explicit
  intake without directional folders.
- Preserved session-to-project recovery and automatic Git text checkpoints,
  plus document, OCR, memory, update, Lifesaver, and formal improvement flows.
