# Lifesaver

Use only when the user asks for diagnosis, recovery, rescue, continuity, or
work preservation.

1. Diagnose before changing anything; prefer read-only checks.
2. Identify the linked project from SessionStart context and read its
   `MASTER.md`. If there is no project, do not create one merely to diagnose.
3. Check the smallest relevant boundary: local files and permissions, bundled
   tool status, network/provider reachability, dependency availability, then
   external service status.
4. Preserve unsaved or at-risk work inside the linked project. Do not create a
   second task, notification, synchronization, or memory system.
5. Keep credentials, private content, and OCR bodies out of logs and records.
6. Stop before destructive work, account changes, paid actions, uncertain
   source-of-truth conflicts, or difficult-to-reverse changes.
7. Report symptom, evidence, diagnosis, action, verification, unresolved
   dependency, and next action. Do not claim recovery without a live check.
