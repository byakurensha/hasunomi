# Memory

`MEMORY/` is portable, user-owned knowledge. It is not project state and it is
not a transcript archive.

1. Search before writing:
   `python3 system/management-tool/management-tool memory search "terms"`.
2. Keep `MEMORY/MEMORY.md` as a concise index. Put detail in
   `MEMORY/notes/` only when needed.
3. Merge, replace, or remove stale and duplicate entries. Add likely future
   search terms.
4. Store only knowledge likely to change future decisions or save meaningful
   work. Never store secrets, raw OCR text, logs, or one-time trivia.
5. Current project status always stays in its `MASTER.md`.
