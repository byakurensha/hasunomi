HASUNOMI / Claude Code VS Code Easy Startup Pack

はじめ方

1. VS CodeでGeneralフォルダを開きます。
2. 公式Claude Codeパネルを開きます。
3. setup と入力します。
4. 言語と3つの作業設定に答えます。環境確認やインストールの前には、
   何を行うかが先に案内されます。

セッションと作業

公式Claude Codeのセッション一覧だけを使います。新しい仕事は新しい
セッションで始め、続きは同じセッションを再開します。Generalフォルダは
開いたままでよく、内部の案件フォルダを自分で選ぶ必要はありません。

一般的な質問には案件を作りません。素材、成果物、ファイル変更、後日の
継続がある時だけ、Claudeが日付付きの内部案件を自動的に作ります。

- MASTER.md：目的、判断、現在地、次の作業
- Materials：取り込んだ素材の保存コピー
- Output：成果物の正本
- Archives：削除せず残す旧版

人が受け取る成果物は、デスクトップのOUTBOXへコピーされます。General内に
専用のINBOXやOUTBOXは作りません。

SHAREBOX

デスクトップのSHAREBOXは、別の環境や別のGeneralセッションと受け渡すための、
1つの一時置き場です。Claudeは案件のOutputから選んだファイルを検証して置き、
選択した項目を現在の案件のMaterialsへ検証して取り込めます。正常に取り込めた
時だけ元を削除し、同名項目は上書きしません。方向や目的が必要なら、申し送りの
Markdownに書きます。方向別フォルダは作らず、SHAREBOXを保管庫にしません。

文書とOCR

同封のA4文書と簡易議事録は1つの標準A4テンプレートを共有し、編集可能な
ODTを標準出力にします。PDFは明示された時だけ作成します。自由デザインを
選んだ場合は、Noto Sans、ODT、LibreOffice、特定の制作方法を強制しません。

同封の「フォトスライド」は、スマホ写真から編集可能な16:9のODPを作ります。
タイトル、細い水平線、ページ番号が入り、横長・縦長の写真1〜2枚を組み合わせる
6種類の配置を使います。写真は標準では切り抜かず、埋め込み用コピーだけ向きを
正規化します。PDFは明示された時だけ作成します。

OCRは元ファイルを変更せず、抽出テキストのMarkdownを作ります。日本語
セットアップはTesseractのjpn+engと、同封テンプレート用のNoto Sans CJK JP
を準備します。英語は英語OCRを準備し、このフォントを必須にしません。
OCRエンジンは後から交換でき、実際に使ったエンジンとフォールバックを
報告します。

メモリ、改善、改修

MEMORYは、好みや再利用したい学びを後から検索できる形で保存します。
案件の現在地はMASTER.mdに置きます。「改善プロセスを実行して」と頼むと、
正式な7段階の見直しを行います。繰り返す問題など、十分な根拠がたまった
場合にも提案しますが、固定回数では強制しません。見直しの結果、変更なし
になる場合もあります。

改善の最後には、軽量な構造チェックが自動で入ります。説明の重複、矛盾、
退役済み経路、常時読み込むファイルの肥大化が見つかった時は、独立した
「改修プロセス」を実行します。改修はROLE、ルール、Skill、マニュアル、実装の
整合性と前後の読込コストを確認し、動いている機能を保ったまま整理します。
毎回の作業後に強制する処理ではありません。

外部プラグイン

Claude Code内蔵の /plugin 画面から、Generalで使う外部プラグインを追加できます。
初期状態では、第三者のマーケットプレイスやプラグインは何も追加しません。
信頼できる配布元だけを登録してください。インストール、更新、無効化、削除は
Claude Codeが管理し、HASUNOMIの製品ファイルには混ぜません。変更後に案内が
出た場合は /reload-plugins を実行します。

更新、Git、バックアップ

更新ZIPは、Claudeに「このZIPを使って、今開いているGENERALを更新して」と
頼みます。更新機能は製品ファイルだけを置き換え、大文字の利用者状態と
ローカル設定を保護します。
Gitは主要なMarkdownの運用ファイル、再利用するMemory、各案件のMASTER.md
だけを、案件完了時または次のセッション開始時に自動記録します。VS Codeの
ソース管理を利用者が操作する必要はありません。Materials、Output、Archivesと
その他のファイルは、テキストであっても普段使っているクラウド同期や
バックアップで保護してください。

復旧

問題がある時は自然な言葉で診断や復旧を頼むとLifesaverを使います。まず
診断し、作業を保全し、別のタスク・通知・メモリシステムは作りません。
